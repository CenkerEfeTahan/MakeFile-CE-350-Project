Contributing
Thank you for considering contributing to this project.

How to contribute
Fork the repository
Create a new branch for your change
Make your changes and test them
Commit with a clear message explaining what you did
Open a pull request
Code style
Try to keep the script simple and easy to read as much as possible. Follow the existing style in the project and avoid unnecessary complexity.

Testing
Before opening a pull request, make sure the script works as expected so try first. It’s especially helpful to test both command-line arguments and piped input.

Issues and suggestions
If you notice a bug or have an idea to improve the tool, feel free to open an issue and describe it briefly.
